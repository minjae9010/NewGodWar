# NewGodWar v0.1.3

NewGodWar의 첫 GitHub 릴리즈입니다. 기존 `0.1.0-SNAPSHOT` 기반에서 팀전 진행, 랜덤 능력, 도박, 설정 GUI, 호환성 검증까지 실제 서버 운영에 필요한 흐름을 크게 다듬었습니다.

## Highlights

- 3팀 기반 Gods War 진행 흐름을 정리했습니다.
- 랜덤 능력 배정, 재추첨, 수동 지정, 도감 GUI를 확장했습니다.
- 팀 심장, 스폰, 중간 참여, 관전, 팀 채팅 등 운영 명령을 보강했습니다.
- 도박 GUI와 보상 아이템, 당첨 멘트 설정을 개선했습니다.
- 부두술사 능력의 연결 피해량과 타격 간격을 설정할 수 있게 했습니다.
- 다크니스가 받는 피해 배율을 설정할 수 있게 했습니다.
- Paper 버전별 스모크 테스트와 GitHub Release 자동화를 추가했습니다.

## 이전 버전 대비 주요 변경점

### 게임 진행

- 빨강, 파랑, 초록 3팀 자동 배정과 수동 배정 명령을 개선했습니다.
- 팀별 스폰과 다이아 심장 등록 기능을 추가하고, 심장 파괴 시 팀 탈락 흐름을 정리했습니다.
- 게임 시작 전 준비 카운트다운, 빠른 시작, 능력 확정 대기 스킵을 지원합니다.
- 진행 중인 게임에 중간 참여할 수 있는 흐름을 추가하고, 탈락 팀 플레이어가 살아있는 팀으로 재참여할 수 있게 했습니다.
- 관전 모드 전환과 관전 해제 명령을 추가했습니다.

### 능력 시스템

- 내장 능력 등록 구조를 확장하고 일반/고급 능력을 추가했습니다.
- 능력 GUI와 능력 도감 GUI를 추가해 현재 능력과 등록된 능력을 더 쉽게 확인할 수 있습니다.
- 능력 재추첨, 랜덤 재배정, 제거, 컷인 보조 명령을 개선했습니다.
- 타깃형 능력 대상 지정과 블록 상호작용 처리를 보강했습니다.
- 능력 쿨타임 처리, 반복 타이머, 우르프 모드 쿨타임 배율을 개선했습니다.
- 부두술사 연결 피해량과 타격 간격을 `abilities.voodoo.damage`, `abilities.voodoo.hit-interval-millis` 설정으로 조정할 수 있습니다.
- 다크니스 피격 피해 배율을 `abilities.darkness.incoming-damage-multiplier` 설정으로 조정할 수 있습니다.

### 팀과 운영 편의

- 팀 색상 스코어보드와 탭 목록/머리 위 이름 Prefix 표시를 보강했습니다.
- 팀킬 방지와 팀 채팅을 지원합니다.
- 관리자용 설정 GUI를 추가해 서버 안에서 주요 게임 설정을 조정할 수 있습니다.
- `/godwar tips`와 게임 시작 준비 안내용 서버 플레이 팁을 추가했습니다.
- 랜덤 배정에서 특정 능력을 제외할 수 있는 블랙리스트 관리 명령을 추가했습니다.

### 도박 시스템

- `/도박`, `/t con` 기반 카지노 GUI를 추가했습니다.
- 일반/타짜 보상 목록을 확장했습니다.
- 보상 아이템 변경, 보상 추가, 당첨 메시지 수정 명령을 개선했습니다.
- 설정 파일에서 보상 확률 가중치와 메시지를 조정할 수 있도록 정리했습니다.

### 호환성과 빌드

- NMS 리플렉션 어댑터 기반 액션바/타이틀 전송 구조를 정리했습니다.
- bStats 의존성을 shadow jar에 relocate하도록 빌드를 개선했습니다.
- 최신 지원 Paper 플러그인 시작 스모크 테스트와 레거시 버전 호환성 매트릭스를 추가했습니다.
- GitHub Actions에서 빌드, 호환성 스모크 테스트, jar 업로드, GitHub Release 생성을 자동화했습니다.

## 지원 버전

- Paper/Spigot 계열 MC 1.7.10부터 MC 1.21.x까지의 레거시 및 최신 서버를 목표로 합니다.
- 최신 지원 Paper 버전에 대한 별도 스모크 테스트를 함께 실행합니다.
- Pre-release / RC 서버는 공식 지원 대상이 아닙니다.

## 다운로드

아래 Assets에서 `NewGodWar-0.1.3.jar`를 다운로드한 뒤 서버의 `plugins` 폴더에 넣어 주세요.

## 검증

- `./gradlew --no-daemon clean build`
- 최신 지원 Paper / Java 21 시작 스모크 테스트
- 레거시 Paper 버전 매트릭스는 별도 호환성 워크플로우로 계속 확인합니다.
