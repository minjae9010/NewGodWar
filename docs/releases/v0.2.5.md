# NewGodWar v0.2.5

NewGodWar v0.2.5는 로비 월드와 게임 월드를 분리해 운영하고, 게임 월드를 백업/복원할 수 있는 월드 관리 기능을 추가한 운영 기능 릴리즈입니다.

## Highlights

- `/godwar world` 명령군을 추가해 월드 생성, 로드, 복사, 이동, 언로드, 삭제, 백업을 관리할 수 있습니다.
- `/godwar setlobby`로 접속 및 게임 종료 후 복귀할 로비 위치를 저장할 수 있습니다.
- `world.game-world`와 `world.reset-game-world-on-stop` 설정으로 게임 시작 전 스냅샷을 만들고 종료 후 게임 월드를 복원할 수 있습니다.
- `normal`, `flat`, `void` 월드 생성 타입을 지원하며 공허 월드 생성을 위한 내부 월드 생성기를 추가했습니다.
- 설정 GUI에 월드 전용 메뉴를 추가하고, 월드 운영 문서를 새로 작성했습니다.
- 게임 종료 시 인벤토리 정리와 로비 복귀 흐름을 설정으로 제어할 수 있습니다.

## 이전 버전 대비 주요 변경점

### 월드 관리 명령

- `/godwar world list`로 로드된 월드와 로드 가능한 월드 폴더를 확인합니다.
- `/godwar world game <world|clear>`로 자동 초기화 대상 게임 월드를 지정하거나 해제합니다.
- `/godwar world create <world> [normal|flat|void]`와 `/godwar world load <world> [normal|flat|void]`로 월드를 생성/로드합니다.
- `/godwar world copy <sourceWorld> <newWorld> [normal|flat|void]`로 기존 월드 폴더를 복사해 새 월드로 로드합니다.
- `/godwar world tp <world> [player]`, `/godwar world lobby [player]`로 월드 이동과 로비 이동을 처리합니다.
- `/godwar world unload <world> [save]`와 `/godwar world delete <world> confirm`으로 월드를 안전하게 정리합니다.
- `/godwar world backup create|list|load`로 `plugins/NewGodWar/world-backups/` 아래 월드 백업을 관리합니다.

### 자동 초기화와 로비

- 게임 월드가 지정되어 있으면 게임 시작 직전에 스냅샷을 만들고, 게임 종료 후 참가자/옵저버를 로비로 이동한 뒤 게임 월드를 복원합니다.
- 로비 위치가 없거나 게임 월드에 플레이어가 남아 있으면 안전을 위해 자동 복원을 건너뜁니다.
- `lobby.teleport-on-join`, `lobby.teleport-on-game-stop`, `game.clear-inventory-on-stop` 설정을 추가했습니다.
- `world.managed-worlds`는 플러그인이 생성/로드/복사한 월드를 기록하고 서버 시작 시 자동 로드에 사용합니다.

### 문서와 GUI

- Wiki에 `월드 관리` 문서를 추가하고 사이드바, 명령어, 설정 문서를 갱신했습니다.
- 설정 GUI에서 월드 자동 저장, 스폰 설정, 난이도, 시작 시간, 게임 월드 지정, 자동 초기화 여부를 조정할 수 있습니다.

## 지원 버전

- Paper/Spigot 계열 MC 1.12.x부터 MC 1.21.x, 그리고 MC 26.1.1부터 26.1.2까지를 지원 대상으로 합니다.
- 최신 지원 Paper 버전에 대한 별도 스모크 테스트를 함께 실행합니다.
- MC 1.11.x 이하 Legacy 서버는 공식 지원 대상이 아닙니다.
- Pre-release / RC 서버는 공식 지원 대상이 아닙니다.
